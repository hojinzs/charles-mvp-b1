# Claude 작업 가이드

## 시작하기 전에

이 프로젝트에 대한 작업을 시작하기 전에, **반드시** 다음 문서를 먼저 읽어주세요:

### 필수 확인 문서

1. **`/docs/SPEC.md`** - 프로젝트 전체 스펙 및 요구사항
   - 프로젝트의 목적과 문제 정의
   - 기술 스택 및 아키텍처 개요
   - 주요 기능 상세 설명
   - 성공 기준 및 제약사항

## 작업 원칙

### 1. 스펙 우선 확인

- 새로운 기능을 추가하거나 수정하기 전에 `/docs/SPEC.md`를 확인하세요
- 요청사항이 스펙의 범위에 포함되는지 검토하세요
- 스펙과 충돌하는 경우 사용자에게 명확히 안내하세요

### 2. 플랜 수립 시 참고사항

- 플랜을 세울 때는 `/docs/SPEC.md`의 다음 섹션을 중점적으로 검토:
  - **3. 범위 정의**: 포함/제외 범위 확인
  - **5. 시스템 아키텍처 개요**: 전체 구조 이해
  - **6. 주요 기능 상세**: 구현 대상 기능의 세부사항
  - **9. 결과 해석 원칙**: 데이터 처리 및 표시 원칙

### 3. 기술 스택 준수

이 프로젝트는 다음 기술 스택을 사용합니다:

- **플랫폼**: Electron (단일 애플리케이션)
- **프론트엔드**: React
- **데이터베이스**: SQLite (로컬)
- **언어**: TypeScript/Node.js

다른 기술 스택의 도입을 고려할 때는 사용자와 상의하세요.

### 4. POC 특성 고려

이 프로젝트는 **POC(Proof of Concept)** 단계입니다:

- 완벽한 완성도보다는 **기술적 검증**이 우선입니다
- 대규모 확장성을 고려하되, 현재는 **기본 동작 검증**에 집중합니다
- UI는 **이해 가능성과 신뢰감**을 우선하며, 디자인 완성도는 차순위입니다
